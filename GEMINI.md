# 프로젝트: 오딘 보스 알리미

## 프로젝트 개요

이 프로젝트는 오딘(Odin)과 같은 게임에서 보스 출현 시간에 대한 알림을 제공하도록 설계된 클라이언트 측 웹 애플리케이션입니다. 사용자는 텍스트 영역에 보스 출현 시간 목록을 입력할 수 있습니다. 이 애플리케이션은 보스 출현 5분 전, 1분 전, 그리고 정확한 출현 시간에 오디오 및 시각적(로그) 알림을 제공합니다. 또한, 현재 보스 목록을 인코딩하여 공유 가능한 짧은 URL(TinyURL API 사용)을 생성하는 기능을 제공하여 사용자가 자신의 설정을 쉽게 공유할 수 있도록 합니다.

**주요 기능:**
*   **보스 목록 관리:** 사용자는 보스 출현 시간과 이름을 입력하고 수정할 수 있습니다.
*   **시간 기반 알림:** 보스 출현 5분 전, 1분 전, 0분 전에 알림이 트리거됩니다.
*   **오디오 알림:** 웹 음성 API(`window.speechSynthesis`)를 활용하여 음성 알림을 제공합니다.
*   **로깅:** 트리거된 모든 알림 및 시스템 메시지 로그를 표시합니다.
*   **공유 가능한 URL:** 다른 사용자를 위해 보스 목록을 미리 채워주는 짧은 URL(TinyURL API를 통해)을 생성합니다.

**사용된 기술:**
*   HTML5
*   CSS3 (내장)
*   바닐라 자바스크립트 (내장)
*   웹 음성 API (`window.speechSynthesis`)
*   TinyURL API (URL 단축용)

## 빌드 및 실행

이 프로젝트는 단일 HTML 파일이며, 특별한 빌드 단계나 서버 측 구성 요소가 필요하지 않습니다.

애플리케이션 실행 방법:
1.  `index.html` 파일을 최신 웹 브라우저(예: Chrome, Firefox, Edge)에서 직접 엽니다.

## 사용법

1.  **보스 목록 편집:** 제공된 텍스트 영역에서 보스 시간과 이름을 수정합니다. 형식은 각 줄에 `HH:MM 보스이름`이어야 합니다.
2.  **알림 시작:** "알림 시작" 버튼을 클릭하여 알림 시스템을 활성화합니다. 버튼 텍스트는 "알림 중지 (실행 중)"으로 변경됩니다.
3.  **알림 중지:** "알림 중지 (실행 중)" 버튼을 클릭하여 알림 시스템을 비활성화합니다.
4.  **공유 링크 생성:** 보스 목록을 수정한 후 "공유 링크 생성 (Short URL)" 버튼을 클릭합니다. 짧은 URL이 아래 입력 필드에 나타납니다.
5.  **공유 링크 복사:** 생성된 URL 옆의 "복사" 버튼을 클릭하여 클립보드에 복사합니다.
6.  **공유:** 복사된 URL을 다른 사람들과 공유합니다. 다른 사람들이 이 URL을 열면, 보스 목록 텍스트 영역이 공유된 데이터로 미리 채워집니다.
7.  **알림 로그:** "알림 로그" 섹션에서 시스템 메시지 및 트리거된 알림을 모니터링합니다.

## 개발 규칙

*   모든 애플리케이션 로직(HTML, CSS, JavaScript)은 `index.html` 파일 내에 포함되어 있습니다.
*   자바스크립트 코드는 다양한 기능 섹션(예: DOM 요소, 전역 변수, 음성 기능, 로깅, 파싱, 알림 확인, 이벤트 리스너)을 나타내는 주석과 함께 구조화되어 있습니다.
*   TinyURL API와 같은 API 호출과 같은 비동기 작업을 처리하기 위해 `async/await`를 사용합니다.
*   API 호출 및 클립보드 작업에 대한 오류 처리가 포함되어 있습니다.
*   애플리케이션은 공유된 보스 목록을 로드하기 위해 URL에서 `data` 매개변수를 확인하여 초기화됩니다.

### Git 커밋 작업 원칙

#### 커밋 제안
- 커밋 제안은 사용자님이 명시적으로 요청하지 않는 이상 제안하지 않는다.

#### 커밋 메시지 생성
- 커밋 메시지는 제목과 본문을 포함하여 영어로 생성한다.
- **참고: 여러 줄 커밋 메시지 작성 시, 임시 파일을 생성하여 메시지를 작성한 후 `git commit -F <파일경로>` 명령을 사용하는 것을 권장합니다. 커밋 완료 후에는 해당 임시 파일을 삭제해야 합니다. 이는 셸 환경에서 여러 줄 메시지 처리 시 발생할 수 있는 문제를 방지합니다.**
- 커밋 메시지 유형:
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `docs`: 문서 변경
  - `style`: 코드 스타일 변경 (포매팅 등)
  - `refactor`: 코드 리팩토링
  - `test`: 테스트 코드 추가/수정
  - `chore`: 빌드 설정, 패키지 업데이트, 기타 잡무성 변경
  - `perf`: 성능 개선
  - `ci`: CI 설정 변경 (예: GitHub Actions, Travis 등)
  - `build`: 빌드 관련 파일 수정 (예: webpack, rollup 설정)
  - `revert`: 이전 커밋 되돌리기
  - `env`: 환경 설정 파일 변경 (.env, config 등)
  - `deps`: 의존성 추가/제거/업데이트

#### 커밋 실행 절차
- 사용자님의 해당 메시지에 대한 명시적인 승인을 받으면, 커밋을 수행할 도구 호출(tool call) 코드 블록을 제시하고 실행한다.
- 커밋이 완료되면 다음 지시를 위해 대기한다.

#### 서브모듈 커밋 원칙
- 서브모듈 포인터를 커밋할 때는 가장 최근 커밋 메시지만 포함하지 말고, 부모 저장소의 마지막 커밋 이후 발생한 모든 서브모듈 커밋 메시지를 빠짐없이 포함해야 한다.
