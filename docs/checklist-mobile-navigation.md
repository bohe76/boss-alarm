#### **3. CSS 스타일링 (`src/style.css`)**
- [x] **(3-1)** 모바일 전용 스타일을 위한 `@media (max-width: 768px)` 미디어 쿼리 블록을 생성한다.
- [x] **(3-2)** 모바일 뷰에서 데스크톱 사이드바(` #sidebar`)와 토글 버튼(` #sidebarToggle`)을 숨긴다.
- [x] **(3-3)** `#bottom-nav`를 화면 하단에 고정하고, CSS 변수 `--bottom-nav-height`를 정의한다.
- [x] **(3-4)** 하단 탭 바의 안전 영역(`env(safe-area-inset-bottom)`)을 `padding-bottom`으로 확보한다.
- [x] **(3-5)** `<main>` 콘텐츠 영역의 `padding-bottom`을 하단 탭 바의 높이와 안전 영역을 고려하여 설정한다.
- [x] **(3-6)** 모바일 '더보기' 메뉴를 위한 `.more-menu-open` 클래스 스타일을 정의하고, 데스크톱 `.sidebar-collapsed`와 충돌하지 않도록 한다.
- [x] **(3-7)** 백드롭 활성화 스타일 (`.sidebar-backdrop.active`)을 정의한다.
- [x] **(3-8)** `body`에 추가될 `is-mobile-view` 클래스에 따른 분기 처리 스타일을 정의한다.

#### **4. JavaScript 로직**
- [x] **(4-1) 뷰포트 감지:** `ResizeObserver`를 사용하여 `<body>`에 `is-mobile-view` 클래스를 동적으로 토글하는 로직을 구현한다.
- [x] **(4-2) 상태 동기화:** `showScreen` 함수가 화면 전환 시, 사이드바와 하단 탭 바 양쪽의 `active` 클래스를 모두 업데이트하도록 수정한다.
- [x] **(4-3) '더보기' 메뉴 이벤트:**
    - [x] '더보기' 버튼 클릭 시 `.more-menu-open`과 `.active` 클래스를 토글하고 `aria-expanded` 속성을 업데이트한다.
    - [x] 백드롭 클릭 시 '더보기' 메뉴를 닫는다.
- [x] **(4-4) 접근성 강화:**
    - [x] '더보기' 메뉴가 열리면 배경 요소들(`main`, `header` 등)에 `inert` 속성을 추가한다.
    - [x] 메뉴 내에서 키보드 초점이 벗어나지 않도록 '초점 가두기(Focus Trap)'를 구현한다.
    - [x] `Escape` 키를 누르면 '더보기' 메뉴가 닫히도록 구현한다.
    - [x] 메뉴가 닫히면 위의 모든 접근성 관련 처리를 원상 복구한다.

#### **5. 최종 검증 (테스트)**
- [ ] **(5-1) 뷰 전환:** 브라우저 창 크기를 조절할 때 데스크톱/모바일 레이아웃이 정상적으로 전환되는지 확인한다.
- [ ] **(5-2) 상태 동기화:** 한쪽 뷰에서 메뉴를 선택하고 다른 뷰로 전환했을 때, 메뉴 활성 상태가 일관되게 유지되는지 확인한다.
- [ ] **(5-3) 모바일 기능:** 하단 탭 바의 모든 버튼(메뉴, 더보기)과 백드롭이 기획대로 동작하는지 확인한다.
- [ ] **(5-4) 접근성:** '더보기' 메뉴가 열렸을 때 초점 가두기, `inert` 속성, `Escape` 키 기능이 모두 정상 동작하는지 확인한다.
- [ ] **(5-5) 레이아웃:** `safe-area`가 있는 기기에서 UI가 겹치거나 잘리지 않는지 확인한다.
