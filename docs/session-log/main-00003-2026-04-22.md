<decisions>
- **[share] URL-safe base64 채택**: 표준 base64의 `+`/`/`/`=` 중 `+`가 form-urlencoded 값에서 공백으로 치환되는 웹 표준 동작이 근본 원인. `encodeURIComponent` 래핑 대신 인코더 자체를 URL-safe로 바꿔 호출부 변경 0.
  - Why: 사용자 스케줄에 한글이 많으면 UTF-8 → base64에서 `+` 확률이 높다. 재현도 1회로 확인됨. 단축 URL 서비스(TinyURL) 경유 시에도 동일 손상이 전파되므로 인코딩 자체에서 방지하는 게 맞다.
  - How to apply: 추후 공유 URL 외 다른 base64 payload를 URL에 실을 때도 동일 전략. `btoa().replace(/\+/g,'-').replace(/\//g,'_').replace(/=+$/,'')` 패턴 복붙.
- **[release] v3.0.1 패치 공지 모달 교체**: `update-notice.json`을 3.0.0 메이저 공지에서 3.0.1 패치 공지로 완전 교체. `hide_update_modal_v{APP_VERSION}` 키 체계 상 3.0.0 숨기기한 사용자도 3.0.1에서 한 번은 모달이 뜨는 것이 의도.
  - Why: 이번 패치가 "공유 링크 깨짐"이라는 사용자 체감 버그 수정이라 모달로 안내할 가치가 있다.
  - How to apply: 마이너 이상은 항상 공지, 패치는 체감형 버그만 공지로. CSS-only/내부 리팩터 패치면 modal 교체 스킵 가능.
- **[release] package-lock.json도 bump 대상**: v3.0.0 관례 유지. package.json만 올리면 `npm install` 시 lock이 자동 갱신되며 diff가 지저분해지므로 수동으로 2곳 선반영.
</decisions>
