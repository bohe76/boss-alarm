# 제안된 추가 개발 기능

## 1. 데스크톱 알림 (Notification API) 및 오디오 음소거 기능

### 개요
현재 오디오 알림은 브라우저가 최소화되거나 백그라운드에 있어도 동작하지만, 시각적인 알림은 제공되지 않습니다. 이 기능은 웹 Notification API를 활용하여 운영체제 수준의 데스크톱 알림을 제공하고, 오디오 알림에 대한 음소거 기능을 추가하여 사용자가 알림 방식을 유연하게 제어할 수 있도록 합니다.

### 상세 내용

#### 1.1. 데스크톱 알림 (Notification API)
*   **목표:** 브라우저 탭이 활성화되어 있지 않거나 다른 애플리케이션을 사용 중일 때도 사용자에게 보스 출현 알림을 시각적으로 전달합니다.
*   **동작 방식:**
    *   `alarm-scheduler.js`에서 알림이 트리거될 때, `speech.js`의 음성 알림과 함께 웹 Notification API를 호출하여 데스크톱 알림을 발생시킵니다.
    *   알림 메시지에는 보스 이름과 남은 시간 등의 정보가 포함됩니다.
    *   사용자에게 최초 1회 알림 권한 요청을 수행합니다.
*   **장점:**
    *   사용자가 소리를 듣기 어려운 환경에서도 알림을 인지할 수 있습니다.
    *   오디오 알림을 놓쳤을 경우 시각적으로 다시 한번 상기시켜 줍니다.
    *   다른 작업을 하느라 브라우저를 보고 있지 않을 때도 중요한 정보를 놓치지 않게 합니다.

#### 1.2. 오디오 알림 음소거 기능
*   **목표:** 사용자가 오디오 알림의 활성화 여부를 직접 제어할 수 있도록 합니다.
*   **동작 방식:**
    *   UI에 오디오 알림을 켜고 끌 수 있는 토글 버튼 또는 체크박스를 추가합니다.
    *   이 설정은 `LocalStorageManager`를 통해 로컬 스토리지에 저장되어 앱 재시작 시에도 유지됩니다.
    *   `speech.js`에서 음성 알림을 실행하기 전에 이 음소거 설정 상태를 확인하여, 음소거 상태일 경우 음성 알림을 건너뜁니다.
*   **장점:**
    *   사용자의 선호도에 따라 알림 방식을 맞춤 설정할 수 있습니다.
    *   오디오 알림이 불필요한 상황(예: 조용한 환경, 다른 오디오를 듣고 있을 때)에서 방해를 줄일 수 있습니다.

### 상호 작용 시나리오
이 두 기능이 함께 제공될 경우, 사용자는 다음과 같은 알림 방식을 선택할 수 있습니다:
*   **오디오 알림만 활성화:** 소리로만 알림을 받습니다. (데스크톱 알림 비활성화, 오디오 알림 활성화)
*   **데스크톱 알림만 활성화:** 음소거 상태이거나 소리를 원치 않을 때 시각적인 알림만 받습니다. (데스크톱 알림 활성화, 오디오 알림 비활성화)
*   **둘 다 활성화:** 오디오와 데스크톱 알림을 모두 받아 알림의 확실성을 높입니다. (데스크톱 알림 활성화, 오디오 알림 활성화)
*   **모두 비활성화:** 알림을 받지 않습니다. (데스크톱 알림 비활성화, 오디오 알림 비활성화)

## 2. 사용자 정의 알림 시간 설정

### 개요
현재 보스 출현 5분 전, 1분 전, 정각에 고정된 알림 시간 대신, 사용자가 직접 원하는 알림 시간을 설정할 수 있도록 하는 기능입니다.

### 상세 내용
*   **목표:** 사용자가 자신의 플레이 스타일이나 선호도에 맞춰 알림을 받을 시간을 유연하게 조정할 수 있도록 합니다.
*   **동작 방식:**
    *   UI에 알림 시간을 추가/삭제/수정할 수 있는 입력 필드 또는 설정 섹션을 제공합니다. (예: "보스 출현 X분 전 알림")
    *   설정된 알림 시간은 `LocalStorageManager`를 통해 로컬 스토리지에 저장되어 앱 재시작 시에도 유지됩니다.
    *   `alarm-scheduler.js`는 설정된 사용자 정의 알림 시간을 기반으로 알림을 트리거합니다.
*   **장점:**
    *   개인의 필요에 맞는 맞춤형 알림 경험을 제공합니다.
    *   특정 보스나 상황에 따라 알림 전략을 변경할 수 있는 유연성을 확보합니다.
