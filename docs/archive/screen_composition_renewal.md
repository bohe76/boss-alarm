# 화면 구성 리뉴얼 제안: 메뉴 기반 다중 화면 레이아웃

기존 단일 화면 대시보드형 구성에서 벗어나, 사용자 경험을 더욱 체계적이고 직관적으로 개선하기 위해 **메뉴 기반의 다중 화면 레이아웃**을 제안합니다. 각 기능 영역을 별도의 화면으로 분리하고, 명확한 내비게이션 메뉴를 통해 사용자가 원하는 정보와 기능에 쉽게 접근할 수 있도록 합니다.

**핵심 원칙:**
*   **기능별 분리:** 유사한 기능들을 그룹화하여 별도의 화면으로 구성합니다.
*   **명확한 내비게이션:** 직관적인 메뉴를 통해 화면 간 이동을 용이하게 합니다.
*   **정보 접근성:** 사용자가 필요한 정보에 빠르게 도달할 수 있도록 합니다.

---

### **1. 전체 레이아웃 구성**

애플리케이션은 크게 **헤더**, **내비게이션 메뉴**, **메인 콘텐츠 영역**, **푸터**로 구성됩니다.

*   **헤더:** 앱의 제목과 전역적인 액션(알람 ON/OFF 토글)을 포함합니다.
*   **내비게이션 메뉴:** 각 기능 화면으로 이동할 수 있는 링크를 제공합니다. (좌측 사이드바)
*   **메인 콘텐츠 영역:** 선택된 메뉴 항목에 따라 해당 기능의 콘텐츠를 표시합니다.
*   **푸터:** 버전 정보 등을 표시합니다.

---

### **2. 헤더**

*   **앱 제목:** "🎙️ 보스 알리미" (좌측 상단, 텍스트 줄바꿈 방지)
*   **알람 ON/OFF 토글:** 우측 상단에 배치하여, 앱의 어느 화면에서든 알람을 켜고 끌 수 있도록 합니다. "ON" 텍스트 SVG 아이콘 버튼 형태로 구현하며, 알람 활성화 시 빨간색, 비활성화 시 진한 회색(#333)으로 표시됩니다.
*   **배경:** 본문과 동일한 #f8f8f8 색상이며, 텍스트 색상은 #111입니다.

### **3. 내비게이션 메뉴**

좌측에 **접을 수 있는(collapsible) 사이드바** 형태로 구현하여, 다음 화면들로 이동할 수 있는 메뉴 항목을 제공합니다. 기본적으로 아이콘과 텍스트 라벨을 모두 표시하며, 사용자가 사이드바를 접을 경우 아이콘만 표시되고 마우스 오버 시 툴팁으로 메뉴 이름을 보여줍니다. 사이드바의 상단 또는 헤더에 위치한 **전용 토글 버튼(햄버거 아이콘)**을 클릭하여 사이드바를 접거나 펼 수 있습니다.

### **4. 메인 콘텐츠 영역**

내비게이션 메뉴에서 선택된 항목에 따라 다음 화면 중 하나가 표시됩니다. 배경색은 #f8f8f8입니다.

#### **4.1. 화면 1: 대시보드**
*   **역할:** 앱의 핵심 정보를 한눈에 파악할 수 있는 요약 화면입니다.
*   **내용:**
    *   **다음 보스:** "다음 보스" 텍스트는 검정색, 굵게, 1.1em 크기로 표시되며 줄바꿈됩니다. 그 아래에 보스 시간, 보스 이름, 남은 시간 정보가 빨간색, 굵게, 1.5em 크기로 표시됩니다.
    *   **다가오는 보스 목록:** 다음 10개의 보스 이름과 각 보스까지 남은 시간을 간략하게 목록 형태로 표시합니다.
    *   **알림 상태 요약:** 현재 알림이 실행 중인지 여부, 다음 알림 예정 시간 등 간략한 알림 상태 정보를 표시합니다.
    *   **최근 알림 로그:** 가장 최근에 발생한 1개의 알림 메시지를 간략하게 표시합니다. (전체 로그는 '알림 로그' 화면에서 확인)
*   **목표:** 사용자가 앱을 열었을 때 가장 먼저 확인하고 싶어 하는 핵심 정보를 즉시 제공하고, 앱의 현재 상태를 빠르게 인지할 수 있도록 합니다.

#### **4.2. 화면 2: 보스 관리**
*   **역할:** 보스 목록을 편집하고 알림을 제어하는 화면입니다.
*   **내용:**
    *   **보스 목록 입력 텍스트 영역:** 현재와 동일하게 보스 시간과 이름을 입력하는 영역입니다. 높이는 400px입니다.
    *   **초기 로딩 시 날짜 자동 추가:** URL에서 보스 목록을 불러오지 않을 경우, 기본 보스 목록에 오늘 날짜가 맨 위에 추가됩니다. 또한, 기본 목록 내에 다음 날로 넘어가는 보스 시간(예: 23:00 이후 01:00)이 감지되면 해당 지점 앞에 내일 날짜가 추가됩니다. (기존 목록에 날짜 항목이 없는 경우에만 적용)
*   **목표:** 보스 목록과 관련된 모든 작업을 한 곳에서 집중적으로 처리할 수 있도록 합니다.

#### **4.3. 화면 3: 알림 설정**
*   **역할:** 모든 알림 관련 설정을 한 곳에 모아 사용자가 쉽게 접근하고 변경할 수 있도록 하는 화면입니다.
*   **내용:**
    *   **고정 알림 설정:** 현재의 고정 알림 목록 및 개별 토글 스위치
*   **목표:** 알림의 종류, 시점, 방식 등 모든 알림 관련 설정을 직관적으로 관리할 수 있도록 합니다.

#### **4.4. 화면 4: 알림 로그**
*   **역할:** 애플리케이션의 모든 시스템 메시지 및 트리거된 알림 내역을 상세하게 확인하는 화면입니다.
*   **내용:**
    *   시스템 메시지 및 트리거된 알림을 표시하는 스크롤 가능한 영역. 높이는 400px입니다.
    *   로그 가시성 토글 스위치가 "알림 로그" 제목 옆에 배치됩니다.
    *   로그 항목의 시간 표기 방식은 `HH:MM:SS`입니다.
    *   로그 항목의 폰트 크기는 0.9em이며, 시스템 폰트 패밀리를 사용합니다.
*   **목표:** 앱의 동작 상태와 알림 내역을 상세하게 확인하고 문제 발생 시 디버깅에 활용할 수 있도록 합니다.

#### **4.5. 화면 5: 도움말**
*   **역할:** 애플리케이션의 사용 방법, 기능 가이드 등을 제공하는 화면입니다.
*   **내용:**
    *   "기능 가이드" 콘텐츠를 직접 표시합니다.
    *   폰트 크기는 0.9em이며, 시스템 폰트 패밀리를 사용합니다.
*   **목표:** 사용자가 앱을 효과적으로 활용할 수 있도록 필요한 정보를 제공하고, 궁금증을 해소할 수 있도록 돕습니다.

#### **4.6. 화면 6: 버전 정보**
*   **역할:** 애플리케이션의 현재 버전 정보와 업데이트 내역을 표시하는 화면입니다.
*   **내용:**
    *   `docs/version_history.txt` 파일의 내용을 기반으로 업데이트 내역을 표시합니다.
    *   폰트 크기는 0.9em이며, 시스템 폰트 패밀리를 사용합니다.
*   **목표:** 사용자가 현재 사용 중인 앱의 버전을 쉽게 확인하고, 어떤 변경 사항이 있었는지 상세하게 파악할 수 있도록 합니다.

#### **4.7. 화면 7: 공유**
*   **역할:** 현재 보스 목록을 인코딩하여 공유 가능한 짧은 URL을 생성하고 관리하는 화면입니다.
*   **내용:**
    *   "공유" 메뉴 항목 클릭 시 자동으로 링크를 생성하고 클립보드에 복사하며, 진행 상황 메시지를 표시합니다.
*   **목표:** 보스 목록 공유 과정을 간소화하고 사용자에게 명확한 피드백을 제공합니다.

**전반적인 디자인 고려사항:**

*   **반응형 디자인:** 다양한 화면 크기(데스크톱, 모바일)에 맞춰 레이아웃이 유연하게 조정되도록 합니다.
*   **간결한 UI:** 불필요한 요소는 제거하고, 중요한 정보와 기능에 집중합니다.
*   **시각적 계층:** 제목, 버튼, 텍스트 등의 크기, 색상, 굵기를 조절하여 정보의 중요도를 시각적으로 표현합니다.
*   **아이콘 활용:** 메뉴 항목이나 버튼, 설정 항목에 적절한 아이콘을 사용하여 직관성을 높입니다. (현재는 기존 아이콘 유지, 알람 ON/OFF 토글 버튼은 "ON" 텍스트 SVG 사용)
*   **일관된 디자인:** 모든 화면에서 일관된 디자인 시스템과 컴포넌트를 사용하여 통일감 있는 사용자 경험을 제공합니다.

이러한 메뉴 기반의 다중 화면 구성은 앱의 확장성을 높이고, 사용자가 복잡한 기능들을 더욱 쉽게 이해하고 제어할 수 있도록 도울 것입니다.
