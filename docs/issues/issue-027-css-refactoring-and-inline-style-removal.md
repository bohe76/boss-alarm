# Issue-027: CSS 리팩토링 및 인라인 스타일 제거

## 1. 개요 (Overview)
현재 프로젝트 전반에 걸쳐 CSS가 여러 파일 (`layout.css`, `components.css`, `screens.css`, `style.dark.css` 등)에 분산되어 있고, HTML 및 JS 코드 내에 **인라인 스타일(Inline Styles)**이 혼재되어 있어 유지보수가 매우 어렵습니다.
특히 `!important` 남발과 `width: fit-content` 등의 속성이 의도치 않은 **사이드 이펙트(레이아웃 틀어짐)**를 유발하고 있어, 이를 구조적으로 개선하는 리팩토링이 시급합니다.

## 2. 문제점 (Problem)
1.  **인라인 스타일 혼재:** HTML 태그 및 JS 코드(`element.style.xxx = ...`) 내에 CSS 속성이 직접 정의되어 있어 외부 CSS 파일의 규칙을 덮어쓰거나 우선순위 충돌을 일으킴.
2.  **분산된 스타일 정의:** 유사한 컴포넌트의 스타일이 여러 CSS 파일에 흩어져 있어 일관성 유지가 어렵고 수정 시 누락 발생.
3.  **사이드 이펙트 발생:** 특정 화면을 위해 수정한 CSS가 다른 화면(보스 스케줄러, 설정, 로그 등)의 레이아웃을 망가뜨리는 현상 반복.
4.  **`!important` 및 `fit-content` 오남용:** 레이아웃 제어를 위해 사용된 강력한 속성들이 예상치 못한 동작(예: 카드 너비가 너무 좁아짐)을 유발함.

## 3. 작업 목표 (Goals)
1.  **인라인 스타일 전면 제거:** HTML/JS 내의 모든 인라인 스타일을 제거하고 CSS 클래스로 이관.
2.  **CSS 구조 일원화 및 모듈화:** 역할에 따라 CSS 파일을 명확히 재정의하고 중복 코드 제거.
3.  **예측 가능한 레이아웃 시스템 구축:** `card-size-xxx` 등 공통 클래스의 동작을 명확히 정의하고, 화면별 예외 처리를 최소화하여 사이드 이펙트 방지.
4.  **다크 모드 일관성 확보:** 리팩토링된 구조가 다크 모드(`style.dark.css`)에도 정확히 반영되도록 동기화.

## 4. 실행 계획 (Action Plan)
1.  **현황 파악:** `grep` 등을 통해 인라인 스타일 사용처 전수 조사.
2.  **인라인 스타일 제거:** JS 동적 스타일링(필수 불가결한 경우 제외)과 HTML 내 `style="..."` 속성을 제거하고 클래스 기반으로 전환.
3.  **CSS 클래스 재정의:** `card-size-standard`, `card-size-list` 등의 너비 및 반응형 동작(Breakpoint 포함)을 재설계.
4.  **화면별 검증:** 보스 스케줄러, 대시보드, 설정, 로그 등 모든 화면에서 레이아웃 깨짐 여부 전수 테스트.
