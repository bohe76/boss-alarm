# Issue-011: 로그 시스템 리팩토링 및 다색상 로깅 기능 구현

## 1. 문제점

현재 애플리케이션의 로그 시스템은 `isImportant` 플래그 하나로만 로그 메시지의 중요도(빨간색 또는 기본색)를 구분하고 있습니다. 이로 인해 다음과 같은 문제가 발생합니다.

*   **의미 불분명:** 다양한 종류의 중요한 이벤트(보스 알림, 시스템 이벤트, 사용자 액션)가 모두 같은 '빨간색'으로 표시되어 로그의 의미를 파악하기 어렵습니다.
*   **사용자 혼란:** 특히 음소거 ON/OFF와 같은 반복적인 사용자 액션까지 빨간색으로 표시되어, 정작 중요한 알림 메시지의 시인성이 저하되고 로그 내용이 혼란스러워집니다.
*   **유연성 부족:** 보스 관련 이벤트, 시스템 상태 이벤트 등 로그의 유형에 따라 다른 색상을 적용하여 시각적 구분을 명확히 할 수 없습니다.

## 2. 목표

`log()` 시스템을 리팩토링하여 로그 유형별로 다른 색상을 적용할 수 있는 기능을 구현하고, 로그 메시지의 시각적 명확성과 사용성을 높입니다.

## 3. 제안하는 변경 사항

### 3.1. 새로운 색상 규칙

다음과 같은 색상 규칙을 도입하여 로그 메시지의 시각적 구분을 명확히 합니다.

*   **빨간색 (`#ea4335`):** 보스와 직접적으로 관련 있는 경우 (예: 보스 출현 임박 알림, 보스 파싱 에러).
*   **청록색 (`#43AA8B`):** 시스템의 상태를 나타내는 경우 (예: 알림 시스템 시작/중지, 자정 초기화, 음소거 변경).
*   **기본 색상 (`#333`):** 위 두 가지에 해당하지 않는 일반적인 정보 및 비치명적 오류 로그.

### 3.2. 구현 상세

1.  **`src/logger.js` 수정:**
    *   `log(message, isImportant)` 함수의 시그니처를 `log(message, logType = 'info')`로 변경합니다.
    *   `logType` 파라미터는 `'boss'`, `'system'`, `'info'`와 같은 문자열 값을 받습니다.
    *   `log()` 함수 내부에서 `logType` 값에 따라 `log-type-boss` (빨간색), `log-type-system` (청록색) 또는 클래스 없음 (기본색) 등의 CSS 클래스를 로그 항목에 추가하도록 로직을 변경합니다.

2.  **`src/style.css` 수정:**
    *   `.log-entry.important` 클래스는 더 이상 사용하지 않거나, 새로운 `log-type-boss` 클래스로 대체합니다.
    *   `.log-type-boss` 클래스에는 빨간색 (`#ea4335`) 스타일을 정의합니다.
    *   `.log-type-system` 클래스에는 청록색 (`#43AA8B`) 스타일을 정의합니다.

3.  **모든 `log()` 호출 지점 수정:**
    *   코드베이스 내에서 `log()` 함수를 호출하는 모든 지점(약 20개 파일, 70회 이상 호출)을 식별하고, 새로운 색상 규칙 및 `logType` 파라미터에 맞춰 호출 방식을 변경합니다.
    *   각 로그 메시지의 내용과 중요도에 따라 `log(메시지, 'boss')`, `log(메시지, 'system')`, `log(메시지, 'info')` 등으로 적절하게 수정합니다.

## 4. 작업 브랜치 제안

이 변경은 코드베이스 전반에 걸쳐 광범위한 수정이 필요하므로, 별도의 피처 브랜치(예: `feature/multi-color-logging`)에서 진행할 것을 제안합니다.
