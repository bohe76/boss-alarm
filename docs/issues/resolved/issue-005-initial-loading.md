# Issue-005: 초기 로딩 시 깜빡임 문제 및 스켈레톤 UI 적용 준비

## 문제점

애플리케이션 초기 로딩 시 CSS가 완전히 적용되기 전에 HTML 콘텐츠가 잠시 노출되는 'Flash of Unstyled Content (FOUC)' 현상이 발생하여 사용자 경험을 저해합니다. 이 문제는 특히 SPA(Single Page Application)에서 빈번하게 발생하며, 사용자가 애플리케이션의 초기 로딩 상태를 불편하게 느끼게 합니다.

## 해결 방안 (해결됨)

사용자 경험 개선을 위해 초기 로딩 시 FOUC를 완화하고, 애플리케이션의 시각적 안정성을 높이기 위한 '스켈레톤 UI' 적용을 준비 중입니다.

### 스켈레톤 UI 개요

스켈레톤 UI는 실제 콘텐츠가 로드되는 동안 사용자에게 페이지의 대략적인 레이아웃을 미리 보여주는 일종의 로딩 인디케이터입니다. 이는 콘텐츠 로딩 시간을 시각적으로 채워주어 사용자에게 로딩이 진행 중임을 명확히 인지시키고, 실제 콘텐츠 로딩 후의 시각적 변화를 최소화하여 더 부드러운 전환 효과를 제공합니다.

### 구현 상세

1.  **스켈레톤 UI 도입:**
    *   `index.html`에 대시보드 화면의 주요 UI 요소에 대한 스켈레톤 플레이스홀더를 추가하여 초기 로딩 중 사용자에게 시각적 피드백을 제공합니다.
    *   스켈레톤 UI 관련 CSS는 `index.html` 내 `<style>` 태그에 인라인 처리하여 로딩 지연 없이 즉시 표시되도록 보장합니다.

2.  **모바일 뷰 Transient UI 문제 해결:**
    *   초기 로딩 시 모바일 뷰에서 잠시 나타났다 사라지던 불필요한 UI 요소(좌측 사이드바, 햄버거 메뉴 토글 버튼, 사이드바 닫기 버튼) 문제를 해결했습니다. `src/style.css`에서 해당 요소들이 모바일 미디어 쿼리 내에서 기본적으로 `display: none;`으로 설정되도록 하여 FOUC(Flash of Unstyled Content)를 방지했습니다.

3.  **모바일 레이아웃 안정화:**
    *   모바일 뷰에서 콘텐츠가 적을 때 푸터가 의도치 않게 상단으로 올라오는 문제를 해결하기 위해 `src/style.css`의 `main` 요소에 `min-height` 속성을 적용했습니다. 이로써 푸터는 항상 화면 하단에 올바르게 위치합니다.

4.  **HTML 구조 및 린팅 환경 개선:**
    *   `index.html`에 존재하던 중복 `</main>` 태그 오류를 수정하여 HTML 구조의 유효성을 확보했습니다.
    *   `eslint.config.js` 설정을 업데이트하여 테스트 코드 린팅 환경을 개선하고 개발 편의성을 높였습니다.
