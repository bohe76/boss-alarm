# Issue-010: 고정 알림 URL 데이터 처리 문제

## 1. 문제점

*   사용자 로컬 스토리지에 저장된 고정 알림과 공유 URL을 통해 전달된 고정 알림 데이터(`fixedData` 파라미터) 간에 충돌 가능성이 존재했습니다.
*   기존 구현은 URL의 `fixedData`가 존재할 경우 로컬의 고정 알림을 덮어쓰는 방식으로 동작하여, 사용자 데이터 손실의 위험이 있었습니다.
*   이는 사용자의 개인적인 설정인 고정 알림의 특성을 고려할 때 적절하지 않은 처리 방식이었습니다.

## 2. 해결 방안

*   **공유 URL 생성 시:** '공유' 기능을 통해 URL을 생성할 때, '고정 알림' 데이터(`fixedData` 파라미터)는 URL에 포함하지 않습니다. 오직 동적 보스 목록(`data` 파라미터)만 포함합니다.
*   **URL 로딩 시:** 앱이 URL을 통해 로드될 때, 만약 `fixedData` 파라미터가 URL에 포함되어 있더라도 해당 데이터는 완전히 무시하고 처리하지 않습니다. 이는 과거 URL과의 호환성을 고려하되, 사용자에게 혼란을 주거나 데이터 손실을 야기하는 것을 방지하기 위함입니다.
*   이로써 고정 알림은 사용자 로컬 스토리지에서만 관리되는 개인적인 설정으로 유지되며, URL 공유 기능과는 완전히 독립됩니다.
