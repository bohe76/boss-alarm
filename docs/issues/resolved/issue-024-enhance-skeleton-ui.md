# Issue-024: 대시보드 레이아웃 최신화에 따른 스켈레톤 UI 고도화

## 1. 개요 (Overview)
기존 정적이고 단조로운 스켈레톤 UI를 현재의 2단 컬럼 대시보드 레이아웃과 일치하도록 고도화하여 사용자 경험을 개선하고 시각적 인지 부하를 줄입니다.

## 2. 문제점 또는 요구사항 (Problem or Requirement)
*   기존 스켈레톤 UI가 과거의 단일 컬럼 레이아웃을 기반으로 하고 있어 로딩 완료 후 실제 콘텐츠로 전환될 때 큰 레이아웃 시프트(CLS)가 발생함.
*   단순한 회색 박스 형태로 애니메이션이 없어 로딩 중인지 멈춘 상태인지 구분이 어려움.
*   실제 대시보드의 소리 설정, 카카오 내보내기 등 주요 컴포넌트의 형태가 반영되지 않음.

## 3. 제안된 해결 방안 (Proposed Solution)
*   **레이아웃 동기화**: `index.html` 내 스켈레톤 HTML 구조를 `.dashboard-layout` 기반의 2단 컬럼 구조로 재설계.
*   **심미성 강화**: `skeleton-loading` (Shimmer) 효과 및 `fadeIn` 애니메이션 도입.
*   **컴포넌트 묘사**: 실제 카드 규격(`.card-standard`)과 아이콘 위치를 고려한 정밀한 스켈레톤 아이템 배치.

## 4. 해결 과정 및 최종 결과
1.  **HTML 수정**: `#dashboard-skeleton` 내부를 실제 대시보드 구조인 좌/우 컬럼 및 소형/표준 카드 조합으로 재구성함.
2.  **스타일 수정**: 
    *   Shimmer 애니메이션 스타일 정의 및 전 영역 적용.
    *   스켈레톤 카드들이 실제 레이아웃 클래스의 영향을 받도록 하여 반응형 대응 완료.
3.  **데이터 바인딩**: `app.js`에서 스켈레톤 제거 시점을 최적화하여 콘텐츠 전환이 부드럽게 일어나도록 함.
4.  **검증**: 임시 코드 동결을 통해 스켈레톤 UI가 실제 대시보드와 정확히 겹치는 것을 확인하고, 애니메이션의 자연스러움을 점검함.
