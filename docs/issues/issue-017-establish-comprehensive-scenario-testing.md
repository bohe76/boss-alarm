# Issue-017: 사용자 시나리오 기반 전체 통합 테스트 시스템 구축

## 1. 개요 (Overview)
* 보스 스케줄러를 포함한 앱의 핵심 기능들이 사용자 시나리오에 따라 유기적으로 동작하는지 코드 레벨에서 자동 검증하는 테스트 체계를 구축합니다.

## 2. 문제점 또는 요구사항 (Problem or Requirement)
* **문제점**: 현재 테스트가 단위 기능(Unit Test) 위주로 파편화되어 있어, 화면 초기화, 탭 전환, 데이터 저장 및 복구 등 여러 모듈이 얽힌 실제 사용자 시나리오에서 발생하는 결함을 사전에 탐지하기 어려움.
* **요구사항**: 
    1. 사용자가 앱에 진입하여 데이터를 입력하고, 탭을 전환하고, 설정을 적용하기까지의 전체 프로세스를 시뮬레이션하는 테스트 코드 추가.
    2. 브라우저를 직접 띄우지 않고도(Headless) CI/CD 환경이나 로컬 개발 단계에서 코드 레벨로 즉시 검증 가능해야 함.
    3. 로컬 스토리지 연동, 이벤트 버스 통신, UI 렌더링이 결합된 통합 시나리오 확보.

## 3. 제안된 해결 방안 (Proposed Solution)
* **Vitest + JSDOM 기반 통합 테스트 강화**:
    * `test/boss-scheduler.scenario.test.js` (가칭) 생성.
    * 실제 `index.html`의 DOM 구조를 최대한 반영한 환경 구축.
    * 사용자의 실제 액션 시나리오(예: 오딘 선택 -> 시간 입력 -> 텍스트 모드 전환 확인 -> 적용 -> 타임테이블 이동)를 하나의 테스트 케이스로 구현.
* **Playwright 도입 검토 (필요 시)**:
    * 실제 브라우저 환경에서 렌더링 및 인터랙션을 완벽하게 보장해야 하는 핵심 경로에 대해 E2E 테스트 도입 계획 수립.
