# 이슈 005: 초기 로딩 시 깜빡임 문제 및 스켈레톤 UI 적용 준비

## 문제점

애플리케이션 초기 로딩 시 CSS가 완전히 적용되기 전에 HTML 콘텐츠가 잠시 노출되는 'Flash of Unstyled Content (FOUC)' 현상이 발생하여 사용자 경험을 저해합니다. 이 문제는 특히 SPA(Single Page Application)에서 빈번하게 발생하며, 사용자가 애플리케이션의 초기 로딩 상태를 불편하게 느끼게 합니다.

## 해결 방안 (준비 중)

사용자 경험 개선을 위해 초기 로딩 시 FOUC를 완화하고, 애플리케이션의 시각적 안정성을 높이기 위한 '스켈레톤 UI' 적용을 준비 중입니다.

### 스켈레톤 UI 개요

스켈레톤 UI는 실제 콘텐츠가 로드되는 동안 사용자에게 페이지의 대략적인 레이아웃을 미리 보여주는 일종의 로딩 인디케이터입니다. 이는 콘텐츠 로딩 시간을 시각적으로 채워주어 사용자에게 로딩이 진행 중임을 명확히 인지시키고, 실제 콘텐츠 로딩 후의 시각적 변화를 최소화하여 더 부드러운 전환 효과를 제공합니다.

### 구현 계획 (준비 중)

1.  **HTML 구조 정의:** `index.html` 내에 대시보드 화면의 주요 UI 요소(예: 다음 보스 정보, 다가오는 보스 목록, 알림 상태, 알림 로그)를 모방하는 스켈레톤 플레이스홀더를 추가합니다. 이 플레이스홀더들은 초기에는 보이도록 설정됩니다.
2.  **CSS 스타일링:** 스켈레톤 UI의 핵심 스타일(레이아웃, 색상, 로딩 애니메이션 등)은 `index.html`의 `<head>` 내 `<style>` 태그에 직접 인라인합니다. 이는 스켈레톤 UI 자체가 로딩 지연 없이 즉시 표시되도록 보장합니다. 추가적인 스켈레톤 관련 스타일은 `src/style.css`에 추가할 수 있습니다. 일반적으로 회색 배경색과 부드러운 애니메이션 효과를 적용하여 로딩 중임을 시각적으로 표현합니다.
3.  **JavaScript 제어:** 애플리케이션의 `initApp()` 함수에서 모든 컴포넌트 초기화 및 데이터 로딩이 완료된 후, 스켈레톤 UI를 숨기고 실제 대시보드 콘텐츠를 표시하도록 로직을 구현합니다.

이슈는 현재 스켈레톤 UI를 적용하기 위한 준비 단계에 있으며, 상세 구현은 추후 진행될 예정입니다.
