# 이슈 001: 반응형 레이아웃 및 모바일 UX 개선

## 1. 문제점

현재 애플리케이션의 반응형 레이아웃은 다양한 화면 크기에서 다음과 같은 문제점을 가지고 있습니다.

- **데스크톱:** 넓은 화면에서 메인 콘텐츠 영역이 과도하게 넓어져 가독성이 떨어지고, 시각적으로 휑한 느낌을 줍니다.
- **모바일:** 왼쪽 사이드바가 가로 너비를 차지하여 실제 콘텐츠 영역이 너무 좁아지며, 메뉴 확장 시 콘텐츠를 밀어내어 사용성이 떨어집니다.

## 2. 제안된 해결책

이 문제들을 해결하기 위해 다음과 같은 세 가지 방안이 논의되었습니다.

### 해결책 A: 콘텐츠 최대 너비 제한 (전체 화면 적용)

- **내용:** 데스크톱의 가독성 문제를 해결하기 위해, 각 화면의 주요 콘텐츠 컨테이너(예: `<section>` 요소)에 `max-width` (예: `960px`)를 적용하고 중앙 정렬합니다. `max-width`보다 좁은 화면에서는 자동으로 너비가 100%로 조절됩니다.
- **기대 효과:** 모든 화면 크기에서 콘텐츠 가독성과 시각적 일관성을 향상시킵니다.

### 해결책 B: 모바일 사이드바 오버레이 (부분 개선안)

- **내용:** 모바일에서 `60px` 아이콘 바는 항상 표시하되, 메뉴 확장 시에만 메뉴 목록이 메인 콘텐츠 위를 덮는 오버레이 형태로 나타나도록 합니다. 메인 콘텐츠는 밀려나지 않습니다.
- **기대 효과:** 메인 콘텐츠 너비를 보존하면서 기존 레이아웃을 최대한 활용합니다.

### 해결책 C: 모바일 하단 탭 바 도입 (전면 개편안)

- **내용:** 모바일에서 왼쪽 사이드바를 완전히 숨기고, 화면 하단에 주요 메뉴 4개와 '더보기(...)' 버튼으로 구성된 고정 '하단 탭 바'를 도입합니다. '더보기'를 누르면 전체 메뉴가 오버레이로 나타납니다.
- **주요 결정사항:** 하단 탭 바에 배치할 4개의 주요 메뉴를 결정해야 합니다. (초기 제안: 대시보드, 보스 설정, 계산기, 알림 로그)
- **기대 효과:** 모바일 화면의 공간 효율성을 극대화하고, 최신 앱 트렌드에 맞는 친숙하고 편리한 UX를 제공합니다.

## 3. 결론 및 다음 단계

- **해결책 A**는 모든 화면의 가독성을 높이기 위한 기본 규칙으로 채택합니다.
- 모바일 네비게이션은 **해결책 C**를 채택하여 사용성을 전면적으로 개선하는 방향으로 진행합니다.
