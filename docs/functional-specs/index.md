# 기능 명세서

이 문서는 애플리케이션의 주요 기능 명세를 상세히 설명합니다. 각 기능은 별도의 문서로 분리되어 있으며, 아래 링크를 통해 접근할 수 있습니다.

## 목차

1.  [애플리케이션 공통 기능](application-common-features.md)
    *   **카카오톡 인앱 브라우저 회피:** 카카오톡 인앱 브라우저에서 발생하는 기능 제한 및 오작동 문제를 해결하기 위해, 인앱 브라우저 환경을 감지하고 사용자에게 외부 브라우저(Android는 자동 리디렉션, iOS는 수동 전환 안내)로 이동하도록 유도합니다.
    *   **PiP 위젯 (Document Picture-in-Picture Widget):** 대시보드의 '다음 보스' 정보 중 보스 이름과 남은 시간을 브라우저 밖 미니 창으로 제공. **보스 수에 따라 창 높이가 동적으로 조절(96px ~ 130px+)되어 최적의 가독성을 유지합니다.**
    *   **지능형 네비게이션 시스템 (v2.2)**: 
        *   **PC**: 레이아웃 흔들림 없는 **호버-확장 오버레이 사이드바** 적용. 햄버거 버튼 없이 아이콘 레일에 마우스를 올리면 메뉴명이 부드럽게 펼쳐집니다. **브레이크포인트 920px를 기준으로 PC/모바일 레이아웃이 전환됩니다.**
        *   **모바일**: 하단 탭 바를 **5개 핵심 메뉴**(대시보드, 시간표, 스케줄러, 계산기, 공유)로 확장하여 엄지손가락 동선 내에서 모든 기능 제어 가능.
    *   **차세대 스켈레톤 UI 엔진 (v2.3)**:
        *   **레이아웃 1:1 매칭**: 대시보드 레이아웃과 동일한 2단 컬럼 구조를 적용하여 로딩 완료 시점에 발생하는 시각적 튐 현상(CLS)을 제거했습니다.
        *   **다이내믹 Shimmer 애니메이션**: 부드러운 빛 흐름 효과를 통해 시스템이 준비 중임을 직관적이고 미려하게 표현합니다.
        *   **컴포넌트 리터럴 묘사**: 다음 보스 하이라이트, 사운드 슬라이더 바 등 실제 UI 요소의 형태를 정밀하게 추상화하여 구현했습니다. **대시보드 및 계산기 카드의 너비는 380px로 표준화되어 시각적 균형을 이룹니다.**
    *   초기 로딩 및 화면 전환
    *   전역 알림 시스템 (음성 및 시각적 알림, **볼륨 조절 기능 포함**)
    *   Local Storage를 통한 설정 및 데이터 저장
    *   TinyURL을 통한 URL 단축 및 공유
    *   EventBus를 통한 모듈 간 통신
### 1.3. DB 기반 스케줄 자동 관리 엔진 (v3.0.0)
- **4-테이블 정규화 DB**: v3.0.0부터 `DB`(로컬 스토리지 기반, `src/db.js`)를 단일 진실 원천으로 사용합니다. `v3_games`, `v3_bosses`, `v3_schedules`, `v3_settings` 4개 테이블로 구성됩니다.
- **자동 GC (가비지 컬렉션)**: `BossDataManager._expandAndReconstruct()`가 `alerted_0min` 완료된 과거 스케줄을 자동 제거합니다. 단, 보스별 최소 1개 항목은 보존됩니다.
- **Future Anchor Keeper**: 미래 인스턴스가 없는 보스의 경우, 젠 주기에 따라 미래 인스턴스를 강제 생성하여 알림 공백을 방지합니다.
- **버전 업데이트 안내 모달 (Update Notification)**:
    *   **버전 체인 트리거**: 새로운 업데이트 시 **숫자로 구성된 `window.APP_VERSION`**과 로컬 스토리지의 노출 기록을 대조합니다. 데이터는 순수 숫자로 관리되며, 사용자에게는 UI 레이어에서 자동으로 "v" 접두사가 붙어 표시됩니다.
    *   **데이터 기반 큐레이션 (Data-driven)**: `update-notice.json`에서 인사말과 요약 목록을 동적으로 로드하여 렌더링하므로, 릴리즈 내용에 따라 항목 수를 유연하게 조절 가능합니다.
    *   **개발자 한마디**: 사용자(개발자)님이 직접 작성한 최신 메시지를 전달합니다.
    *   **영구 차단 제어**: '다시 보지 않기' 완료 시 해당 버전에 대해서는 더 이상 모달을 띄우지 않아 사용자의 흐름을 방해하지 않습니다.

3.  [보스 시간표 (Timetable)](boss-management.md)
    *   보스 목록을 **카드 리스트** 또는 **테이블(표)** 형태로 선택하여 볼 수 있습니다.
    *   **형식 전환 버튼**: '카드 보기' 또는 '표로 보기' 버튼을 통해 레이아웃을 즉시 전환할 수 있습니다. **표준화된 32px 높이의 토글 버튼 규격을 사용합니다.**
    *   '다음 보스' 필터 토글을 통해 현재 시간 이후의 보스만 필터링하여 볼 수 있습니다.
    *   **고정 알림 병합 표시**: `settings.md`의 고정 알림(`v3_settings`)이 보스 알림과 함께 시간표에 통합 표시됩니다. `timetable.js`의 `_expandFixedAlarmsInRange()`가 고정 알림을 `{type:'boss', isFixed:true, ...}` shape으로 확장하여 4경로(일별/표/카드/대시보드)에 병합합니다.
    *   **시간표 내보내기(Export)**:
        *   **실시간 프리뷰**: 내보내기 옵션(날짜, 포함 내용, 스타일) 변경 시 배경 UI가 즉시 업데이트되는 WYSIWYG 환경 제공.
        *   **텍스트 내보내기**: 선택된 필터 조건에 맞춰 텍스트를 생성하여 클립보드에 복사.
        *   **이미지 내보내기**: `html2canvas`를 활용하여 카드 또는 표 스타일의 고해상도 이미지 저장.
        *   **모바일 최적화 정책**: PC 화면과 무관하게, 이미지는 항상 **1단 레이아웃(Single Column)**으로 생성하여 모바일(카카오톡 등) 가독성을 극대화합니다.
        *   **동기화된 디자인**: 별도 렌더링 컨테이너를 사용하지만, 메인 화면과 **동일한 CSS 클래스**를 사용하여 디자인 일관성을 완벽히 유지합니다.
        *   **안정성 가드**: 이미지 생성 중 버튼 비활성화(연타 방지) 및 자동 새로고침 일시 중단 기능 적용.
4.  [보스 스케줄러 (Boss Scheduler)](boss-scheduler.md)
    *   게임 선택 드롭다운 및 동적 보스 입력 필드 렌더링 (폼 기반 단일 입력, 텍스트 모드 제거됨)
    *   `boss-presets.json` 기반의 정밀한 보스별 젠 주기(Interval) 매칭 및 출현 시간 계산
    *   **앱 초기화 시 `syncPresetsToDb()` 자동 호출**: `loadBossSchedulerData()` 내에서 프리셋 DB 동기화가 수행됩니다 (`v3_games`, `v3_bosses` 테이블 갱신).
    *   **하이브리드 입력 최적화**: PC(가로 배치)와 모바일(레이블 포함 세로 배치) 기기별 최적화 레이아웃 및 비고 확장(25자).
    *   "보스 시간 업데이트" 클릭 시 `DB.upsertSchedule()`을 통해 `v3_schedules` 테이블에 저장 후 `BossDataManager._expandAndReconstruct()` 실행.
    *   특수 보스(+12h) 및 **'침공' 보스 필터링**: 침공 보스는 오직 당일 일정에만 포함되며 익일 확장 시 자동 배제.
    *   커스텀 보스 목록 관리 및 젠 주기(Interval) 개별 설정 지원.
