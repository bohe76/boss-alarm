# 기능 명세: 보스 타이머 계산기

## 1. 서론
본 문서는 기존 보스 알리미 웹 애플리케이션에 통합될 "보스 타이머 계산기" 기능에 대한 기능 요구사항을 설명합니다. 이 기능은 사용자가 남은 시간 지속 시간을 입력하여 보스 출현 시간을 빠르게 확인할 수 있는 도구를 제공하는 것을 목표로 합니다.

## 2. 사용자 인터페이스 (UI)
### 2.1. 내비게이션
- "젠 계산기"라는 레이블의 새 메뉴 항목이 사이드바 내비게이션에 추가됩니다.
- 메뉴 항목은 "보스 설정" 바로 다음인 세 번째 위치에 배치됩니다.
- 메뉴 항목에는 계산기 SVG 아이콘이 함께 표시됩니다.

### 2.2. 계산기 화면
- "젠 계산기" 메뉴 항목이 선택되면 전용 화면이 렌더링됩니다.
- 이 화면에는 다음이 포함됩니다:
    - "남은 시간" 입력 필드.
    - "보스 출현 시간" 표시 영역.

## 3. 기능
### 3.1. 입력
- "남은 시간" 입력 필드는 `HH:MM` 또는 `HH:MM:SS` 형식의 시간 지속 시간을 허용합니다.
- `SS`(초) 부분이 생략된 경우(예: `HH:MM` 형식), 묵시적으로 `00`초로 처리됩니다.
- 입력 필드는 `input` 이벤트 리스너를 사용하여 실시간으로 계산을 트리거합니다.

### 3.2. 계산 로직
- 입력 필드의 값이 유효한 `HH:MM` 또는 `HH:MM:SS` 형식과 일치할 때:
    - 애플리케이션은 입력 문자열을 시, 분, 초로 파싱합니다.
    - 파싱된 지속 시간은 현재 시스템 시간에 추가됩니다.
    - 결과로 생성된 미래 시간은 "보스 출현 시간"으로 간주됩니다.
- 입력 형식이 유효하지 않거나 불완전한 경우, "보스 출현 시간" 표시 영역은 비어 있거나 적절한 메시지(예: "잘못된 형식")를 표시해야 합니다.

### 3.3. 출력
- "보스 출현 시간"은 명확하고 사용자 친화적인 형식(예: `HH:MM:SS`)으로 표시됩니다.
- 입력이 유효한 형식과 일치하는 경우, 사용자가 입력할 때 디스플레이가 동적으로 업데이트됩니다.

## 4. 기술 세부 사항
### 4.1. 새 모듈: `src/calculator.js`
- 핵심 계산 로직을 캡슐화하기 위해 새 JavaScript 모듈인 `src/calculator.js`가 생성됩니다.
- 이 모듈은 `calculateBossAppearanceTime(remainingTimeString)`와 같은 함수를 내보내며, 이 함수는 남은 시간을 문자열로 받아 계산된 출현 시간(또는 유효하지 않은 입력 표시)을 반환합니다.

### 4.2. 통합
- `index.html`: 내비게이션을 업데이트하고 새 화면 컨테이너를 추가합니다.
- `src/dom-elements.js`: 새 메뉴 항목, 입력 필드 및 표시 영역에 대한 DOM 참조를 추가합니다.
- `src/ui-renderer.js`: `renderCalculatorScreen()`을 구현하고 `renderScreen()`을 업데이트하여 새 화면을 처리합니다.
- `src/event-handlers.js`: 내비게이션 및 입력 필드에 대한 이벤트 리스너를 추가하고, 계산을 위해 `src/calculator.js`를 호출합니다.
- `src/style.css`: 새 UI 요소에 대한 스타일을 추가합니다.

## 5. 오류 처리
- 유효하지 않은 입력 형식은 계산 오류를 방지하고 사용자에게 명확한 피드백을 제공하여 정상적으로 처리됩니다.
- 극단적인 경우(예: 출현 시간을 먼 미래로 미루는 매우 긴 남은 시간)도 고려해야 하지만, 일반적인 보스 타이머의 경우 문제가 될 가능성은 낮습니다.

## 6. 의존성
- `src/calculator.js`는 표준 JavaScript Date 객체 및 문자열 조작에 의존합니다.
- `src/event-handlers.js`는 `src/calculator.js` 및 `src/ui-renderer.js`에 의존합니다.
- `src/ui-renderer.js`는 `src/dom-elements.js`에 의존합니다.
