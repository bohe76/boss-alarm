# 화면 구성 리뉴얼 제안: 메뉴 기반 다중 화면 레이아웃

기존 단일 화면 대시보드형 구성에서 벗어나, 사용자 경험을 더욱 체계적이고 직관적으로 개선하기 위해 **메뉴 기반의 다중 화면 레이아웃**을 제안합니다. 각 기능 영역을 별도의 화면으로 분리하고, 명확한 내비게이션 메뉴를 통해 사용자가 원하는 정보와 기능에 쉽게 접근할 수 있도록 합니다.

**핵심 원칙:**
*   **기능별 분리:** 유사한 기능들을 그룹화하여 별도의 화면으로 구성합니다.
*   **명확한 내비게이션:** 직관적인 메뉴를 통해 화면 간 이동을 용이하게 합니다.
*   **정보 접근성:** 사용자가 필요한 정보에 빠르게 도달할 수 있도록 합니다.

---

### **1. 전체 레이아웃 구성**

애플리케이션은 크게 **헤더**, **내비게이션 메뉴**, **메인 콘텐츠 영역**, **푸터**로 구성됩니다.

*   **헤더 (Header):** 앱의 제목과 전역적인 액션(도움말)을 포함합니다.
*   **내비게이션 메뉴 (Navigation Menu):** 각 기능 화면으로 이동할 수 있는 링크를 제공합니다. (좌측 사이드바 또는 상단 탭 형태로 구현 가능)
*   **메인 콘텐츠 영역 (Main Content Area):** 선택된 메뉴 항목에 따라 해당 기능의 콘텐츠를 표시합니다.
*   **푸터 (Footer):** 버전 정보 등을 표시합니다.

---

### **2. 헤더 (Header)**

*   **앱 제목:** "오딘 보스 알리미" (좌측 상단)
 *   **알람 on/off 토글:** 우측 상단에 배치하여, 앱의 어느 화면에서든 알람을 켜고 끌 수 있도록 합니다. SVG 아이콘 버튼 형태로 구현하며, 알람 상태에 따라 색상으로 구분합니다.
 
 ### **3. 내비게이션 메뉴 (Navigation Menu)**
 
 좌측에 **접을 수 있는(collapsible) 사이드바** 형태로 구현하여, 다음 화면들로 이동할 수 있는 메뉴 항목을 제공합니다. 기본적으로 아이콘과 텍스트 라벨을 모두 표시하며, 사용자가 사이드바를 접을 경우 아이콘만 표시되고 마우스 오버 시 툴팁으로 메뉴 이름을 보여줍니다. 사이드바의 상단 또는 헤더에 위치한 **전용 토글 버튼(예: 햄버거 아이콘 또는 화살표 아이콘)**을 클릭하여 사이드바를 접거나 펼 수 있습니다.
 
 *   **대시보드 (Dashboard)**
 *   **보스 관리 (Boss Management)**
 *   **알림 설정 (Notification Settings)**
 *   **알림 로그 (Alarm Log)**
 *   **버전 정보 (Version Info)**
 *   **공유 (Share)**
 *   **도움말 (Help)**
 
 ### **4. 메인 콘텐츠 영역 (Main Content Area)**
 
 내비게이션 메뉴에서 선택된 항목에 따라 다음 화면 중 하나가 표시됩니다.
 
 #### **4.1. 화면 1: 대시보드 (Dashboard)**
 *   **역할:** 앱의 핵심 정보를 한눈에 파악할 수 있는 요약 화면입니다.
 *   **내용:**
     *   **실시간 보스 카운트다운:** 가장 가까운 보스까지 남은 시간을 크고 명확한 글씨로 `HH:MM:SS` 형식으로 표시합니다. (예: **발키리: 01:23:45 남음**)
     *   **다가오는 보스 목록:** 다음 2~3개의 보스 이름과 각 보스까지 남은 시간을 간략하게 목록 형태로 표시합니다.
     *   **알림 상태 요약:** 현재 알림이 실행 중인지 여부, 다음 알림 예정 시간 등 간략한 알림 상태 정보를 표시합니다.
     *   **최근 알림 로그:** 가장 최근에 발생한 2~3개의 알림 메시지를 간략하게 표시합니다. (전체 로그는 '알림 로그' 화면에서 확인)
 *   **목표:** 사용자가 앱을 열었을 때 가장 먼저 확인하고 싶어 하는 핵심 정보를 즉시 제공하고, 앱의 현재 상태를 빠르게 인지할 수 있도록 합니다.
 
 #### **4.2. 화면 2: 보스 관리 (Boss Management)**
 *   **역할:** 보스 목록을 편집하고 알림을 제어하며, 공유 및 프리셋을 관리하는 화면입니다.
 *   **내용:**
     *   **보스 목록 입력 텍스트 영역:** 현재와 동일하게 보스 시간과 이름을 입력하는 영역입니다.
     *   **주요 액션 버튼:**
         *   (이전 공유 및 복사 버튼은 '공유' 메뉴로 이동)
     *   **프리셋 기능 (제안 기능):**
         *   "프리셋 선택" 드롭다운: 저장된 보스 목록 프리셋을 선택하여 불러옵니다.
         *   "현재 목록 프리셋으로 저장" 버튼: 현재 텍스트 영역의 보스 목록을 프리셋으로 저장합니다.
 *   **목표:** 보스 목록과 관련된 모든 작업을 한 곳에서 집중적으로 처리할 수 있도록 합니다.
 
 #### **4.3. 화면 3: 알림 설정 (Notification Settings)**
 *   **역할:** 모든 알림 관련 설정을 한 곳에 모아 사용자가 쉽게 접근하고 변경할 수 있도록 하는 화면입니다.
 *   **내용:**
     *   **데스크톱 알림 토글 (제안 기능):** "데스크톱 알림 사용" (켜기/끄기)
     *   **오디오 알림 토글 (제안 기능):** "오디오 알림 사용" (켜기/끄기, 음소거 기능)
     *   **사용자 정의 알림 시간 설정 (제안 기능):**
         *   "알림 시간 추가" 버튼
         *   현재 설정된 알림 시간 목록 (예: 10분 전, 5분 전, 1분 전, 0분) 및 각 항목 옆에 "삭제" 버튼
     *   **고정 알림 설정:** 현재의 고정 알림 목록 및 개별 토글 스위치
 *   **목표:** 알림의 종류, 시점, 방식 등 모든 알림 관련 설정을 직관적으로 관리할 수 있도록 합니다.
 
 #### **4.4. 화면 4: 알림 로그 (Alarm Log)**
 *   **역할:** 애플리케이션의 모든 시스템 메시지 및 트리거된 알림 내역을 상세하게 확인하는 화면입니다.
 *   **내용:**
     *   시스템 메시지 및 트리거된 알림을 표시하는 스크롤 가능한 영역.
     *   "로그 지우기" 버튼 (선택 사항)
     *   로그 필터링/검색 기능 (선택 사항)
 *   **목표:** 앱의 동작 상태와 알림 내역을 상세하게 확인하고 문제 발생 시 디버깅에 활용할 수 있도록 합니다.
 
 #### **4.5. 화면 5: 도움말 (Help)**
 *   **역할:** 애플리케이션의 사용 방법, 기능 가이드, 버전 기록 등을 제공하는 화면입니다.
 *   **내용:**
     *   **탭 인터페이스:** "기능 가이드" 탭과 "버전 기록" 탭을 제공합니다.
     *   **기능 가이드:** 앱의 주요 기능과 사용 방법을 설명하는 콘텐츠를 표시합니다. (예: `docs/feature_guide.txt` 내용)
     *   **버전 기록:** 앱의 업데이트 내역을 표시하는 콘텐츠를 표시합니다. (예: `docs/version_history.txt` 내용)
     *   **FAQ/문제 해결:** 자주 묻는 질문이나 일반적인 문제 해결 방법을 포함할 수 있습니다. (선택 사항)
 *   **목표:** 사용자가 앱을 효과적으로 활용할 수 있도록 필요한 정보를 제공하고, 궁금증을 해소할 수 있도록 돕습니다.
 
 #### **4.6. 화면 6: 버전 정보 (Version Info)**
 *   **역할:** 애플리케이션의 현재 버전 정보와 업데이트 내역을 표시하는 화면입니다.
 *   **내용:**
     *   **현재 버전:** `vX.X.X` 형식으로 현재 애플리케이션의 버전을 크게 표시합니다.
     *   **업데이트 내역:** `docs/version_history.txt` 파일의 내용을 기반으로, 각 버전별 주요 업데이트 내용을 목록 형태로 표시합니다.
 *   **목표:** 사용자가 현재 사용 중인 앱의 버전을 쉽게 확인하고, 어떤 변경 사항이 있었는지 상세하게 파악할 수 있도록 합니다.
 
 #### **4.7. 화면 7: 공유 (Share)**
 *   **역할:** 현재 보스 목록을 인코딩하여 공유 가능한 짧은 URL을 생성하고 관리하는 화면입니다.
 *   **내용:**
     *   **공유 링크 생성 버튼:** 클릭 시 공유 프로세스를 시작합니다.
     *   **진행/결과 모달:**
         *   링크 생성 중: "링크가 생성중 입니다. 잠시만 기다려 주세요..." 메시지 표시.
         *   링크 생성 완료: "클립보드에 복사 되었습니다." 메시지 표시 및 "확인" 버튼 활성화.
         *   "확인" 버튼 클릭 시 모달 닫기.
     *   **생성된 링크 표시 (선택 사항):** 생성된 짧은 URL을 텍스트 형태로 표시하여 사용자가 직접 복사할 수도 있도록 합니다.
 *   **목표:** 보스 목록 공유 과정을 간소화하고 사용자에게 명확한 피드백을 제공합니다.
 
 ### **5. 푸터 (Footer)**
 
 *   **저작권 정보:** (선택 사항)

---

**전반적인 디자인 고려사항:**

*   **반응형 디자인:** 다양한 화면 크기(데스크톱, 모바일)에 맞춰 레이아웃이 유연하게 조정되도록 합니다.
*   **간결한 UI:** 불필요한 요소는 제거하고, 중요한 정보와 기능에 집중합니다.
*   **시각적 계층:** 제목, 버튼, 텍스트 등의 크기, 색상, 굵기를 조절하여 정보의 중요도를 시각적으로 표현합니다.
*   **아이콘 활용:** 메뉴 항목이나 버튼, 설정 항목에 적절한 아이콘을 사용하여 직관성을 높입니다. (Heroicons 사용 예정)
*   **일관된 디자인:** 모든 화면에서 일관된 디자인 시스템과 컴포넌트를 사용하여 통일감 있는 사용자 경험을 제공합니다.

이러한 메뉴 기반의 다중 화면 구성은 앱의 확장성을 높이고, 사용자가 복잡한 기능들을 더욱 쉽게 이해하고 제어할 수 있도록 도울 것입니다.
