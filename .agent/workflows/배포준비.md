---
description: 새로운 버전을 릴리즈하기 위한 버전 관리 작업을 수행합니다. (푸시 제외)
---

1. 사용자에게 업데이트할 버전 넘버(예: 2.17.0)를 입력받습니다.
2. `GEMINI.md`의 **[업데이트 안내 모달 컨펌 절차]**에 따라 사용자에게 개발자 인사말을 질문합니다.
3. `GEMINI.md`의 **[릴리즈 노트 작성 원칙]**에 명시된 JSON 구조와 유형을 준수하여 릴리즈 노트를 생성하고 사용자에게 최종 확인을 받습니다.
4. 승인 시 다음 파일들을 `GEMINI.md`의 **[버전 관리]** 규칙(숫자 관리, UI 태그 사용 등)에 따라 일괄 수정합니다.
   - `index.html`, `app.js`, `ui-renderer.js`, `version_history.json`, `update-notice.json`, `unreleased_changes.md`
5. `/검증` 워크플로우를 호출하여 최종 상태를 확인합니다.
6. 완료 보고 후 대기합니다.
