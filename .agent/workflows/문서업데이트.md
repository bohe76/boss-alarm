description: 코드의 물리적 변화와 문서의 논리적 설명을 100% 동기화하여 기술적 무결성을 유지합니다.
---

1. **[코드-문서 갭 분석]** `git diff` 및 코드 전수 조사를 통해 마지막 문서 업데이트 이후의 함수 시그니처, 데이터 구조(SSOT), 전역 상수, 핵심 로직의 물리적 변화를 식별합니다.
2. **[상호 참조 무결성 체크]** 식별된 코드 변화가 `GEMINI.md`에 명시된 핵심 문서 리스트 각각에 미치는 연쇄적 영향을 파악합니다. (예: 모듈 내부 로직 수정이 데이터 흐름 문서에도 영향을 주는지 확인)
3. **[정밀 동기화 수행]**
   - 문서 내의 용어를 실제 코드의 변수/함수명과 100% 일치시킵니다.
   - 추상적인 기능 설명을 현재 코드의 실제 제어 흐름(if/else, 로직 순서 등)을 반영한 기술적 명세로 교체합니다.
   - 다이어그램 등 시각적 요소보다는 텍스트의 정확도와 기술적 밀도에 집중합니다.
4. **[설계 의도 및 Rationale 기록]** 코드만으로는 파악하기 어려운 설계 결정 이유나 예외 처리의 배경 등을 관련 문서의 원칙이나 상세 섹션에 보강합니다.
5. **[동기화 보고]** 어떤 코드의 변화가 어떤 문서의 어느 섹션에 어떻게 반영되었는지 구체적인 매칭 리스트를 작성하여 보고하고 대기합니다.
